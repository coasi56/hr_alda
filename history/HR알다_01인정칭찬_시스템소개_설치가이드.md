# HR알다 — 시스템 소개 & 설치 가이드

> Slack 연동 HR 인정/칭찬 자동화 시스템  
> 비개발자 HR 담당자를 위한 완전 자동화 가이드

---

## 1. HR알다란?

HR알다는 Slack을 기반으로 하는 HR 인정/칭찬 자동화 시스템입니다. HR 담당자가 코드 없이 Claude Code의 도움을 받아 직접 구축한 시스템으로, 조직 내 긍정적인 문화를 만들고 구성원들의 기여를 가시화합니다.

### 1-1. 핵심 기능

| 기능 | 설명 |
|------|------|
| 이모지 기반 칭찬 전송 | Slack 메시지에 알다 이모지(칭찬/신뢰/주도성/원팀)를 달면 자동으로 집계 |
| 주간 한도 관리 | 1인당 주 10개 한도, 초과 시 DM으로 안내 |
| 자기 칭찬 방지 | 본인 메시지에 이모지 달 시 자동 차단 및 DM 안내 |
| 슬래시 커맨드 | /alda_left, /alda_check, /alda_board |
| 관리자 대시보드 | 웹 기반 대시보드에서 전체 현황, 이모지별 통계, Top 5, 이력 확인 |
| 메시지 원문 연결 | 칭찬받은 메시지 내용 미리보기 및 Slack 원문 바로가기 링크 |

### 1-2. 기술 스택

- **백엔드**: Node.js + Express + Slack Bolt for JS
- **프론트엔드**: React + Vite + Chart.js
- **데이터베이스**: PostgreSQL (Render 무료 플랜)
- **백엔드 배포**: Render.com (무료 플랜)
- **프론트엔드 배포**: Vercel (무료 플랜)
- **소스코드 관리**: GitHub
- **Slack 연동**: Slack Bolt, Event Subscriptions, Slash Commands

> ⚠️ Render PostgreSQL 무료 플랜은 30일 후 삭제됩니다. 장기 운영 시 유료 플랜($7/월) 전환을 권장합니다.

## 2. 사전 준비사항

### 2-1. 필수 계정 및 환경

- GitHub 계정 (https://github.com)
- Render.com 계정 (https://render.com)
- Vercel 계정 (https://vercel.com)
- Slack 워크스페이스 (유료 플랜 권장)
- Google Cloud 계정 (Google Sheets/Calendar 연동 시 필요)
- Node.js 설치 (https://nodejs.org) — v18 이상
- Git 설치 (https://git-scm.com)
- VS Code 설치 (https://code.visualstudio.com)
- Claude Code 설치 (VS Code 확장 프로그램)

### 2-2. Slack App 생성

1. https://api.slack.com/apps 접속
2. "Create New App" → "From scratch" 클릭
3. App Name: HR알다 (또는 원하는 이름)
4. 워크스페이스 선택 후 "Create App"
5. OAuth & Permissions → Bot Token Scopes 추가

#### 필요한 Bot Token Scopes

```
channels:history
channels:read
chat:write
chat:write.public
commands
emoji:read
files:write
im:write
reactions:read
users:read
users:read.email
im:history
```

#### Event Subscriptions 설정

활성화 후 다음 Bot Events 추가:
- `reaction_added`
- `reaction_removed`
- `message.im`

Request URL (배포 후 설정):
```
https://[백엔드 Render URL]/slack/events
```

#### Slash Commands 설정

| 커맨드 | 설명 |
|--------|------|
| /alda_left | 이번 주 잔여 발송 개수 확인 |
| /alda_check | 내가 받은 칭찬 내역 조회 |
| /alda_board | 전체 Top 5 현황 |
| /notice | 공지 예약 발송 |

각 커맨드 Request URL:
```
https://[백엔드 Render URL]/slack/events
```

#### Interactivity & Shortcuts 설정

Interactivity를 ON으로 설정하고 Request URL 입력:
```
https://[백엔드 Render URL]/slack/events
```

#### Slack 이모지 등록

4개의 커스텀 이모지를 워크스페이스에 등록합니다:

| 이모지 | 설명 |
|--------|------|
| `:알다-칭찬:` | 칭찬 이모지 (노란/주황 계열) |
| `:알다-신뢰:` | 신뢰 이모지 (파란 계열) |
| `:알다-주도성:` | 주도성 이모지 (보라 계열) |
| `:알다-원팀:` | 원팀 이모지 (초록 계열) |

> ⚠️ 이모지 이름을 변경할 경우, 코드 내 ALDA_EMOJIS 설정도 함께 변경해야 합니다.

## 3. 설치 및 배포

### 3-1. 코드 다운로드

```bash
git clone https://github.com/coasi56/hr_alda.git
cd hr_alda
```

### 3-2. 환경변수(.env) 설정

`backend/.env` 파일 생성:

```env
# Slack
SLACK_BOT_TOKEN=xoxb-...
SLACK_SIGNING_SECRET=...

# JWT & Admin
JWT_SECRET=[랜덤 문자열 32자 이상]
ADMIN_EMAIL=[관리자 이메일]
ADMIN_PASSWORD=[관리자 비밀번호]

# Database
DATABASE_URL=[Render PostgreSQL Internal URL]

PORT=3000
NODE_ENV=production
```

JWT_SECRET 생성:
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### 3-3. GitHub에 코드 올리기

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/[계정명]/[레포명].git
git push -u origin main
```

> ⚠️ .env 파일과 google-credentials.json은 절대 GitHub에 올리지 마세요. .gitignore에 포함되어 있는지 확인하세요.

### 3-4. Render PostgreSQL 생성

1. https://dashboard.render.com 접속
2. "New +" → "PostgreSQL" 클릭
3. Name: hr-alda-db, Region: Oregon (US West), Plan: Free
4. "Create Database" 클릭
5. 생성 완료 후 "Internal Database URL" 복사
6. backend/.env의 DATABASE_URL에 붙여넣기

### 3-5. Render 백엔드 배포

1. Render 대시보드 → "New +" → "Web Service"
2. GitHub 연결 → 레포지토리 선택
3. Name: hr-alda-backend
4. Build Command: `cd backend && npm install`
5. Start Command: `cd backend && npm start`
6. Environment Variables 추가 (.env 내용과 동일)
7. "Create Web Service" 클릭

### 3-6. Vercel 프론트엔드 배포

1. https://vercel.com 접속 → GitHub 연결
2. "Add New..." → "Project" → 레포지토리 선택
3. Root Directory: `frontend`
4. Environment Variables: `VITE_API_URL = https://[Render 백엔드 URL]`
5. "Deploy" 클릭

`frontend/vercel.json` 파일 (SPA 라우팅용):
```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/" }
  ]
}
```

### 3-7. Slack URL 최종 업데이트

백엔드 배포 완료 후 Slack App 설정에서 모든 URL을 업데이트합니다:
- Event Subscriptions → Request URL
- Slash Commands → 각 커맨드 Request URL
- Interactivity & Shortcuts → Request URL

모든 URL:
```
https://[백엔드 Render URL]/slack/events
```

```
GET https://[백엔드 URL]/
```


### 5-2. 자주 발생하는 문제

| 증상 | 해결 방법 |
|------|-----------|
| 이모지 집계가 안 됨 | Slack Event Subscriptions URL 확인, 봇이 채널에 초대되어 있는지 확인 |
| 슬래시 커맨드 오류 | Slash Commands URL이 배포 URL로 업데이트되어 있는지 확인 |
| 대시보드 로그인 안 됨 | Render 환경변수의 ADMIN_EMAIL/PASSWORD 확인 |
| CORS 오류 | app.js의 CORS origin에 프론트엔드 URL이 포함되어 있는지 확인 |
| 배포 후 데이터 초기화 | DATABASE_URL 환경변수가 Render에 설정되어 있는지 확인 |
| 페이지 새로고침 시 404 | frontend/vercel.json 파일이 있는지 확인 |
